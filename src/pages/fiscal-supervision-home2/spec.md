# 财会监督2.0首页 - 版本2

## 📋 业务与功能

### 1.1 页面定位与核心目标
> 作为系统 L1 首页，承担“系统感建立、业务主题分发、公共协同触达、全局态势提示、基础支撑可达、轻频操作承接”的综合职责，避免入口堆砌与结构说明化，呈现真实政务系统首页体验。

### 1.2 功能清单
- 顶部系统区：系统名称、主题导航（按“日常 / 专项”切换）、全局搜索、消息中心、我的待办、用户入口
- 全局摘要（可扫读）：监督主题数、问题总数、未整改问题、预警事项、整改完成率、考评进度
- 主题导航区（主视觉）：四个一级主题域（日常监督、专项整改、专项检查、财会考评），按列展示，点击展开下级主题列表
- 公共协同：公告/共享/下载摘要、我的待办预览、消息提醒
- 动态/政策/公示/服务：摘要列表（弱化，不抢主视觉）
- 基础支撑入口：法规库、规则库、机构库、人才库、案例库、档案/数据（伴随小型状态摘要）
- 高频轻操作：问题填报、整改审核、查看预警、即席分析（按钮形式，首页级轻操作）

### 1.3 交互要点
- 主题切换：分组 Tab（“日常监督 / 专项监督”）切换后刷新主题卡片集
- 主题卡片：点击进入主题工作台；状态 Pill 支持悬停提示；卡片 Hover 提升
- 轻操作：按钮直接触发对应动作或进入对应事项页
- 全局搜索：触达跨主题检索（接口占位）
- 事件与动作：通过 Axure API 对接 onNavigate/onQuickAction/refreshData 等

## 🧱 布局与结构

- 页面内容宽度：1920 设计规范，内容容器 max-w-1760（左右安全边距各 80px），居中
- 顶部系统区（方案B）：品牌栏（政务深蓝渐变） + 导航行
  - 导航行左侧：主导航：首页 / 财会监督 / 政策法规 / 公开公示 / 财会资讯
  - 导航行右侧：主题分组切换（“日常主题/专项主题”）+ 快捷操作（问题填报/查看预警）
  - 顶部工具（品牌行右侧）：消息中心、我的待办、全局搜索、用户
- 主内容区：按区域纵向排布的白色容器卡（rounded-2xl + 蓝色浅边框）
  - 首屏上部并排布局：全局监督概览 与 公共协同 1:1 等比并列
    - 全局监督概览：标题带 + 6 个轻卡（2/3/6 响应）
    - 公共协同：标题带 + Tab + 列表（单列）
  - C 主题导航区：四列布局，每个主题域为一个独立卡片，包含一级主题标题和可展开的下级主题列表
  - 主题域：日常监督（蓝色）、专项整改（红色）、专项检查（橙色）、财会考评（绿色）
  - D 基础支撑与资源配置：左入口卡网格 + 右摘要计数
- 快捷操作：不再默认展示（如需恢复，放置在最下层通栏）
- 阅读节奏：
  - 第一屏：单行总览条 → 主题分发（主视觉）
  - 第二屏：高频操作 → 信息区列表 → 资源入口
- 响应式策略：
  - ≥1280：9/3 两栏
  - 1024-1279：8/4 两栏
  - <1024：单列，优先主题与操作

### 3.2 模块比例与权重
- 主题区权重 > 全局摘要 > 公共协同 ≈ 动态/政策 > 基础支撑 > 轻操作
- 卡片尺寸：主题卡 1×1 单元为基准；摘要卡更紧凑；基础支撑以链接式清单呈现

## 🎨 视觉规范

### 4.1 设计规范来源
**本设计基于**：
- [ ] 用户提供的设计规范
- [x] 主题设计系统：`/src/themes/ufsp-sky/`（globals.css 与 Token）
- [x] 内置设计指导：`/assets/docs/设计指导（简约）.md`

说明：选择稳重清爽的蓝系基调，卡片化但克制；视觉主次通过字号/色彩/留白控制，而非堆叠装饰。

### 4.2 自定义设计要点（配色与层次）
- 主色阶：
  - Primary：`#0F3D8A`（主）、Secondary：`#1456B8`、Subtle：`#EEF5FF`
  - 背景：`#F5F8FF`（页面底色，减少灰感）
- 中性色：`#0F172A / #334155 / #64748B / #E2E8F0 / #FFFFFF`
- 功能色：红 `#DC2626`、黄 `#F59E0B`、蓝 `#2563EB`
- 边框体系：主容器 `#D9E6FF`，模块边框 `#E6EFFF`
- 卡片层次：白底 + 蓝系浅边框 + 轻阴影（仅用于主容器/主题卡 Hover）
- 状态 Pill：浅底 + 高对比文字（红/黄/蓝）
- 顶栏：深蓝渐变，导航项使用透明白底选中态
- 总览条：单行小组件，避免 KPI 卡片化外观

### 4.3 组件状态
- 默认：浅蓝底/白卡片
- Hover：卡片阴影加深、标题强调色
- Selected/Active：Tab 高亮
- Loading：骨架屏

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）
| 事件名称 | Payload 类型 | 触发时机 | 说明 |
|---------|-------------|---------|------|
| `onNavigate` | `string` | 进入主题/资源/详情 | 路由或容器跳转 |
| `onQuickAction` | `string` | 点击轻操作 | 进入对应事项页 |

### 5.2 动作列表（actionList）
| 动作名称 | Params 类型 | 参数说明 | 功能描述 |
|---------|------------|---------|---------|
| `refreshData` | `string` | 可选标识 | 刷新首页数据 |

### 5.3 变量列表（varList）
| 变量名称 | 类型 | 默认值 | 说明 |
|---------|-----|-------|------|
| `active_category` | `string` | `daily` | 当前主题分组 |

### 5.4 配置项列表（configList）
| 配置项名称 | 类型 | 默认值 | 说明 |
|----------|-----|-------|------|
| `title` | `string` | `财会监督系统` | 系统标题 |
| `showOverview` | `boolean` | `true` | 是否显示全局摘要 |

### 5.5 数据项列表（dataList）
无外部注入，使用演示数据；对接真实接口时保持字段约定。

## ✅ 设计取舍说明
- 主体组织：以“业务主题分发”为主视觉，通过分组 Tab 展现系统主题全貌，避免入口墙。
- 被弱化：全局摘要仅小卡提示；动态/政策/公示放摘要列表；基础支撑以链接+数字呈现；导航二级内容折叠到门户二层。
- 被合并：公告/共享/下载合并为“公共协同”，统一弱化展示；我的待办在此区做预览。
- 不放首页：完整业务闭环、复杂流程图、层级标签、重型分析图；避免干扰分发与阅读节奏。
