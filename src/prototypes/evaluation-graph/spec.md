# 评价图谱 - 规格文档

## 1. 页面定位与核心目标

- 页面名称：评价图谱
- 页面定位：面向财会监督考评场景的关系浏览页，用于查看“政策条例 - 评价指标 - 监督主题”之间的直接关系与二跳承接。
- 核心目标：
  - 像企查查关系图一样，让用户先抓住一个焦点对象，再清晰看到上下游。
  - 强化点击联动，让“选中谁、关联谁、下一步看什么”一眼可见。
  - 避免整页大图谱带来的卡顿与视觉负担，改为按当前焦点按需展示关系。

## 2. 功能清单

- 焦点对象切换：
  - 支持在政策条例、评价指标、监督主题三类对象间切换浏览。
  - 左侧清单点击后，主画布与右侧详情同步更新。
- 关系浏览器：
  - 中心展示当前焦点节点。
  - 左右展示直接关联对象。
  - 底部展示二跳关联对象，帮助继续钻取。
- 观察视角：
  - 支持“全链路 / 政策缺口 / 高风险指标”三个视角。
  - 视角切换时，只刷新当前可浏览对象与问题提示。
- 联动详情：
  - 展示当前焦点的说明、状态、口径、关联数量与治理建议。
- 重点提示：
  - 对高风险指标、缺少制度映射的对象进行显式提示。

## 3. 交互要点

- 点击左侧对象：
  - 中心焦点切换。
  - 上下游关系卡立即联动。
  - 右侧说明与建议同步切换。
- 点击关系卡：
  - 直接将该卡片提升为新的中心焦点。
- 视角切换：
  - “全链路”展示全部对象。
  - “政策缺口”优先展示缺少制度依据或规则补充不足的指标。
  - “高风险指标”优先展示高风险指标，并以其为默认焦点。
- 无整屏自由拓扑拖拽：
  - 采用固定关系浏览器布局，优先保证性能和清晰度。

## 4. 内容规划

### 4.1 信息架构

- 顶部：
  - 页面标题
  - 页面说明
  - 视角切换
- 左侧：
  - 统计概览
  - 浏览类型切换
  - 当前对象列表
- 中部：
  - 关系浏览器主画布
  - 二跳关联区
- 右侧：
  - 焦点对象详情
  - 治理建议
  - 重点风险与缺口

### 4.2 数据来源与关键字段

当前为页面内静态示例数据。核心字段包括：

- 政策条例：
  - `id`、`name`、`level`、`owner`、`summary`、`indicatorIds`
- 评价指标：
  - `id`、`name`、`score`、`risk`、`formula`、`summary`、`gap`、`policyIds`、`topicIds`
- 监督主题：
  - `id`、`name`、`owner`、`progress`、`summary`、`indicatorIds`

## 5. 布局与结构

- 页面采用 `左栏 + 中央关系浏览器 + 右栏` 三段式布局。
- 左栏宽度约 `300px`，中部自适应，右栏宽度约 `360px`。
- 中部关系浏览器采用固定三段：
  - 上游关系列
  - 中心焦点卡
  - 下游关系列
- 二跳关联放在中心区底部，避免信息挤压主关系。
- 页面需优先保证桌面端流畅；窄屏时改为纵向堆叠。

## 6. 视觉规范

- 设计来源：沿用 `ufsp-sky` 主题。
- 视觉方向：
  - 清冷蓝灰底色，关系高亮使用亮蓝。
  - 风险提示使用暖橙/红，避免全页高饱和。
  - 整体观感接近企业关系浏览器，而非图表仪表盘。
- 组件状态：
  - 默认态：白底卡片、弱边框、轻阴影。
  - 悬停态：描边增强、轻微上浮。
  - 焦点态：蓝色描边、强化阴影、中心放大。
  - 风险态：高风险指标显示暖色风险签。

## 7. 实现说明

- 技术栈：React + Tailwind CSS + 项目公共 TopBar。
- 关键状态：
  - `focusMode`：观察视角
  - `browseType`：当前浏览对象类型
  - `selectedNode`：当前中心焦点对象
- 性能策略：
  - 不渲染整张全量图谱。
  - 仅渲染当前焦点对象的直接关系与二跳关系。
- 验收要求：
  - 通过 `node scripts/check-app-ready.mjs /prototypes/evaluation-graph`
