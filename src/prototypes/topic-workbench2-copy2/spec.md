# 日常监督工作台 - 规格文档 (Spec)

## 0. 工作流卡片区 (Workflow Card Area)
### 核心原则
- **风格统一**: 保持无边框设计，与首页及整体页面样式语言对齐。
- **紧凑布局**: 优化卡片高度，标签单行显示，移除多余的连接箭头。
- **直接导航**: 移除点击卡片弹出悬浮窗的逻辑，改为直接进入对应功能模块。

### 卡片样式
- **默认态**: 
  - 无固定进入箭头，视觉更清爽。
  - 卡片宽度恢复为 `240px`，标题字号恢复为 `14px`，保持视觉协调。
  - 卡片高度压缩至 `110px` 左右，减少内部留白，结构更紧凑。
  - 标签 (Tag) 强制一行显示，不换行。
- **Hover态**: 
  - 卡片轻微上浮 (`-translate-y-1`)。
  - 阴影增强 (`shadow-md`)。
  - 右下角滑入显示进入箭头。
- **节点名称**: 
  - 1. 预警下发
  - 2. 人工抽查
  - 3. 疑点填报
  - 4. 疑点认定
  - 5. 整改填报
  - 6. 整改确认
  - 7. 系统抽查

### 独立功能按钮
- **全流程查看**: 
  - 位置：工作流区域右侧（末尾）。
  - 样式：轻量化的圆形按钮（直径 `40px`），内部包含流程图标。
  - 交互：Hover 时图标放大，点击弹出“业务全流程视图”悬浮弹窗。

## 0.1 业务全流程视图 (Full Process)
- **定位**：专题级宏观流程导航页，快速理解流程关系与直达功能页。
- **结构**：
  - 顶栏维持系统风格；
  - 左侧保留“工作台 + 节点菜单”，顺序与工作流卡片一致；
  - 右侧为简化流程图：仅节点关系、流转方向、阶段分组；
  - 左侧竖向阶段带：配置段/疑点阶段/整改阶段，弱化样式，仅作分组辅助。
- **节点**：
  - 起点（并列）：预警下发、人工抽查
  - 疑点阶段：疑点填报 → 疑点认定
  - 整改阶段：整改填报 → 整改确认 → 系统抽查
- **交互**：
  - 点击节点：直接进入对应功能列表页；
  - 悬停节点：出现轻量提示（名称、说明、办理角色、阶段、上下游、进入提示）；
  - 角色相关性：与当前用户相关的节点采用克制的高亮，其他弱化。
- **视觉**：
  - 与工作台一致的轻量风格；
  - 箭头简洁，不做繁重连线；
  - 节点尺寸适中，信息收敛。

## 1. 评价体系分析区 - 规格文档 (Spec)

## 1. 核心目标
构建一个支持“分层钻取分析”的交互模型，清晰表达评价体系的层级结构、分层分析及左右联动逻辑。
页面核心语义为：左侧雷达图展示一级维度，右侧四个页签围绕该一级维度下的二级指标展开分析。

## 1.1 展示口径（分值 vs 业务值）
- **评分结果（得分）**：统一使用“分”，例如“综合得分：89分”“二级指标得分：90分”，不得使用“%”。
- **业务指标原始值（本期值/标准值）**：按指标真实单位展示（如“90% / 92%”“1.8倍 / 2.0倍”“15天”），不得误写为“分”。
- **三级指标页**：必须同时展示“指标得分（分）”与“本期值/标准值（业务单位）”，并保持趋势图口径一致（评分趋势=分）。

## 2. 交互模型：三层递进分析
页面采用“左侧导航/总览 + 右侧详情分析”的联动布局。

### 第一层：宏观维度层 (Macro Level)
- **左侧**: 雷达图展示一级评价类型（如：预算保障、执行支付、合规纪律等）的综合评分。
- **默认态**:
  - 首次进入不预选任何一级维度，保持“一级维度总览”。
  - 右侧显示“一级概览”与评价体系整体说明，不显示单维度下钻按钮。
  - 雷达图仅保留 hover 反馈，不出现默认选中高亮。
- **交互**: 
  - 点击维度名称、点位或雷达区域后直接进入第二层（该维度的二级指标雷达图）。
  - 右侧同步切换为该维度的四个页签分析（维度概览/类型拆解/闭环建议/趋势概览）。
- **右侧页签**:
  - 维度概览：维度说明、当前得分、风险判断、关键二级指标依据。
  - 类型拆解：展示该维度下二级指标清单（名称、得分、风险等级、环比变化）。
  - 闭环建议：基于二级指标短板输出“问题识别→整改建议→建议动作→整改周期”。
  - 趋势概览：展示该维度近6期趋势、平均值和趋势说明。

### 第二层：类型下指标层 (Indicator Level)
- **左侧**: 雷达图切换为展示当前一级维度下的多个二级指标评分。
- **交互**:
  - 点击二级指标名称、点位或雷达区域直接进入第三层单指标分析。
  - 支持通过“返回图2/返回一级概览”快速回退。
- **右侧页签**:
  - 维度概览：维度说明、当前得分、风险等级、3个关键二级指标依据。
  - 类型拆解：完整二级指标结构清单（名称、得分、风险等级、环比变化）。
  - 闭环建议：问题识别→原因分析→整改建议→建议动作→整改周期。
  - 趋势概览：近6期折线趋势、平均值与趋势说明。

### 第三层：具体指标层 (Detail Level)
- **左侧**: 展示当前选中二级指标的趋势图。
- **右侧页签**:
  - 指标概览：同维度二级指标对比并支持切换当前指标。
  - 指标分析：当前指标定义、得分、结果判定。
  - 异常概览：该维度内异常指标与风险提示。
  - 闭环建议：针对当前指标的闭环治理建议。
  - 趋势分析：当前指标的时间序列变化情况。

### 地方政府债务专题：三级页专属要求
- 第三级页采用“结果头部 + 规则与计算逻辑 + 政策命中依据 + 动作区”的结构，不再将信息平均切成左右静态卡片。
- 左侧需压缩为“焦点摘要栏”：
  - 评分视图中，指标切换放在当前结果上方，作为先选择指标、再查看规则结果的前置操作。
  - 指标切换文案使用“同二级指标切换”，当前选中指标不重复展示得分，只保留状态与权重辅助识别。
  - 当前结果仅保留指标名称、得分、状态、权重等必要焦点信息，不重复承载业务结果、参考阈值、评分标准等正文内容。
  - 权重放在当前结果卡右侧，利用右侧空位承载辅助信息。
  - 采用悬停查看方式补充说明：悬停得分查看评分依据，悬停权重查看设权原因。
- 标题文案统一：
  - “同组指标切换”改为“指标切换”
  - “规则与计算逻辑”改为“规则定义”
  - “政策命中依据”改为“政策依据”
- 右侧需强化“结论驱动”：
  - 顶部先给出当前指标处于何种规则区间、是否需要优先处置。
  - 中部拆分为“规则与计算逻辑”“政策命中依据”两个模块。
  - 未形成明确业务闭环前，不展示“查看关联问题”“生成整改任务”“加入重点关注”等动作按钮。
  - 不常驻展示“业务结果 / 参考阈值”，避免将业务原始值误读为评分结果。
  - 规则定义、政策依据采用固定外框和固定上下内边距；内部内容区独立滚动，滚到底仍保留底部留白，外层卡片底部圆角必须完整可见。
- 统计口径属于评价体系规则，不允许在结果页手动切换；评分视图顶部展示“统计口径：月度”并筛选单月，趋势视图顶部展示“统计口径：月度范围”，分别选择起始月份和结束月份；季度口径时分别选择起始季度和结束季度。
- 增加“查看趋势 / 返回评分”入口：
  - 趋势视图沿用现有下钻层级：一级指标趋势 → 二级指标趋势 → 三级指标趋势。
  - 趋势周期跟随当前指标规则口径；月度指标展示月度趋势，季度指标展示季度趋势，年度指标展示年度趋势。
  - 趋势视图只展示规则得分变化，不混入业务原始值，避免百分比或金额被误读为评价得分。
  - 趋势视图左侧上方为“指标下钻”或三级“指标切换”，下方只展示本期得分与较上期变化，不展示小趋势图。
  - 趋势视图右侧不重复展示大标题，不再固定展示统计期间卡片；说明文案和统计卡片均基于顶部自定义范围筛选联动展示。
- 整体视觉延续无边框、轻阴影、浅色分层风格，重点通过背景、留白和色块而不是重描边表达层级。

## 3. 视觉与布局要求
- **面包屑导航**: 顶部显示“综合分析 / 一级维度 / 二级指标”，支持快速回退。
- **返回逻辑**: 提供“返回上一级”按钮。
- **概览返回入口**: 在图表区标题右侧提供“返回一级概览”显式入口，快速回到默认总览态。
- **区域对齐**: 综合分析区默认高度与右侧待办区保持对齐，超出内容在区域内滚动。
- **留白一致**: 左侧图表区在一级总览与二级指标视图下保持一致的底部留白。
- **图表优化**: 
  - 解决文字遮挡：通过标签自动换行、调整雷达图半径和边距实现。
  - 选中态反馈：当前选中项标签高亮，雷达图点位高亮。
  - 默认关闭一级/二级雷达图悬浮提示框，避免遮挡图面。
  - 隐藏雷达图原生文字，仅保留透明可点击覆盖标签层，标签默认不换行。
  - 一级维度说明列表改为紧凑卡片网格，支持直接点击进入维度详情。
- **页签样式**: 一级维度详情页签与标题同一行，采用待办区同款胶囊样式并居右对齐。
- **主从关系**: 左侧为分析对象选择区，右侧为详情内容展示区，保持强联动。

## 4. 技术实现
- **数据结构**: `data.ts` 中定义 `Level1_Dimension`, `Level2_Indicator`, `Level3_IndicatorDetail` 嵌套模型。
- **状态管理**: 
  - `currentLevel`: 当前层级 (1, 2, 3)。
  - `selectedL1`: 当前选中的一级维度对象。
  - `selectedL2`: 当前选中的二级指标对象。
- **图表库**: ECharts。
- **UI 组件**: React + Tailwind CSS + Ant Design Icons。

## 5. 智能分析模块
- 顶部提供时间范围筛选（最近7天/最近30天/本月/本季度/自定义）与分析生成时间展示。
- 左侧使用可点击分析结论卡片，包含结论标题、摘要、标签、来源报告与报告入口。
- 结论卡片支持悬停预览来源报告信息（报告名称、生成时间、涉及维度、涉及指标、对象范围）。
- 点击结论或“查看分析报告”进入报告详情弹层，展示结论概述、影响范围、趋势、触发规则、制度依据、整改建议。
- 右侧改为规则引用列表，展示规则名称、触发依据、状态，并支持查看规则详情。
- 左右联动：悬停/选中结论高亮对应规则；点击规则反向定位关联结论。
