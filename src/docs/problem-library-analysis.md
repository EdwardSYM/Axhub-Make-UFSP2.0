# 问题库功能分析与专项监督评价指标提炼

分析时间：2026-05-07  
分析对象：财会监督系统中“问题库”菜单目录及其相关前端功能点  
分析边界：仅基于登录后的菜单结构、前端路由、页面字段和接口命名进行只读分析，未执行新增、删除、分发、下发、送审、审核、导出等会改变业务数据的操作。

## 1. 结论摘要

“问题库”不是单一的数据查询页面，而是围绕问题整改闭环建立的一组业务功能，覆盖模板配置、问题清单或台账建立、分发录入、责任落实、整改更新、审核上报、督办、销号、明细查询、典型案例沉淀等环节。

从专项监督评价体系角度看，问题库可以提供四类核心评价数据：

1. 问题发现数据：问题数量、问题金额、问题来源、问题领域、区划、单位、项目、问题类型。
2. 整改过程数据：分发、录入、送审、审核、退回、上报、督办、撤销、流程日志。
3. 整改成效数据：已整改金额、正在整改金额、整改完成时间、预计完成时间、是否销号、是否超期、是否上传佐证。
4. 治理提升数据：问责信息、典型案例、制度完善、责任处室和责任人落实情况。

建议将问题库指标纳入专项监督评价体系时，避免只看“问题数量少就是好”。问题数量和金额既可能反映风险暴露，也可能反映发现能力。更适合采用“发现覆盖、整改效率、整改质量、督办闭环、治理成效”五类指标组合评价。

## 2. 问题库菜单结构

系统菜单中“问题库”位于“财会监督模块菜单”下，主要包含以下功能组：

| 功能组 | 主要功能点 | 业务定位 |
|---|---|---|
| 台账模板管理 | 台账模板新增、编辑、启用、停用、导入、导出、表单视图设计、栏目配置、数据来源配置 | 统一问题和整改数据采集口径 |
| 专项问题整改 | 工作台账管理、录入、审核、上报、整改督办、整改下发、整改更新、整改审核、整改上报、整改明细查询 | 面向专项检查类问题的整改闭环 |
| 监管问题整改 | 问题清单管理、整改问题录入、整改问题上报、整改督办、整改下发、整改更新、整改审核、整改上报 | 面向监管发现问题的整改闭环 |
| 审计问题整改 | 台账建立、责任处下发、市财政下发、区财政下发、整改信息填报、汇总上报、整改信息审核、复核、评价局督办 | 面向审计问题的整改闭环 |
| 案例管理 | 典型案例申报、审核、发布、案例明细查询 | 将已整改或有代表性问题沉淀为案例资产 |

此外，系统中还存在“年度重点信息检查”“监控处理单”等相关菜单，也包含问题下发整改、问题整改反馈、疑点核查、违规整改等能力。这些不是本次“问题库”主菜单的核心范围，但可作为专项监督评价体系的数据补充来源。

## 3. 功能点拆解

### 3.1 台账模板管理

台账模板管理用于定义问题库的数据结构和采集规范，是后续指标可信度的基础。

主要功能点：

| 功能点 | 说明 |
|---|---|
| 模板新增与编辑 | 配置台账模板名称、分类、业务分类、专项检查领域、年度、密级、备注等 |
| 表样上传 | 上传台账模板表样文件，支持从表样解析字段 |
| 栏目设计 | 配置标题列、数据列、计算列、上级栏目、列类型、数据类型、数值格式、小数位、千分位等 |
| 表单视图设计 | 配置字段标签、必填、是否显示、所属分组、展示形式、数据来源 |
| 信息项分组 | 包括问题信息项、整改信息项、公共信息项 |
| 模板状态管理 | 支持启用、停用、删除、移动、批量编辑、导入、导出 |

可沉淀的数据对象：

| 数据对象 | 关键字段 |
|---|---|
| 模板 | 模板名称、业务分类、专项领域、年度、密级、状态 |
| 栏目 | 栏目名称、列类型、数据类型、计算公式、是否显示、是否必填 |
| 表单字段 | 标签名、字段分组、展示形式、数据来源、枚举或要素 |

评价价值：

1. 可用于评价各专项是否建立了标准化问题台账。
2. 可用于衡量问题数据完整性和可比性。
3. 可作为跨专项、跨地区分析的字段规范基础。

### 3.2 专项问题整改

专项问题整改主要面向专项检查领域，流程从台账分发录入到整改更新、审核、上报、销号和明细查询。

主要功能点：

| 环节 | 功能点 |
|---|---|
| 台账分发录入 | 工作台账管理、工作台账录入、工作台账审核、工作台账上报 |
| 角色覆盖 | 评价局、市财政、区县、省直、牵头处、归口处、市州领域、区县领域 |
| 整改更新 | 整改督办管理、整改下发管理、整改情况更新、整改情况审核、整改情况上报 |
| 查询归集 | 整改明细查询，支持未销号、已销号、全部等状态视角 |
| 日志追踪 | 流程日志、整改日志、督办日志 |

页面字段体现的核心业务信息：

| 类别 | 字段示例 |
|---|---|
| 问题来源 | 来源类型、自查、复查、责任单位 |
| 基本信息 | 专项检查领域、区划、项目名称、所涉单位、问题具体描述 |
| 问题类型及金额 | 合计、新增地方政府隐性债务、不合规 PPP 模式新增、违规使用专项债券资金、专项债券资金闲置浪费等 |
| 整改情况 | 整改措施、整改情况、已整改金额、整改完成时间、正在整改金额、预计完成时间、备注 |
| 联系信息 | 联系人、联系电话、备用联系人、备用联系电话 |
| 佐证材料 | 整改报告、签批附件、问题清单附件、上传时间、附件类型 |
| 流程状态 | 待提交、已提交、待审核、已审核、被退回、流程中、已办结 |
| 督办信息 | 已督次数、本轮督办开始时间、本轮整改期限、本轮整改截止时间、本轮是否超期、本轮督办说明 |

评价价值：

1. 能衡量专项检查问题的整改速度和质量。
2. 能识别高风险领域、金额集中领域和长期未销号问题。
3. 能评价责任单位、责任处室、牵头处的响应和审核效率。

### 3.3 监管问题整改

监管问题整改与专项问题整改流程类似，但以“问题清单”为核心对象，菜单中带有监管问题整改标识。

主要功能点：

| 环节 | 功能点 |
|---|---|
| 清单分发录入 | 问题清单管理、整改问题录入、整改问题上报 |
| 清单管理 | 新增、编辑、分发、撤销、删除、按部门、按单位、按处室、按区划分发 |
| 整改更新 | 整改督办、整改下发、整改情况更新、审核、上报 |
| 权限角色 | 评价局、责任处、市财政、市州领域、区县领域、省直、归口处等 |

评价价值：

1. 可用于评价监管发现问题是否及时分发到责任主体。
2. 可用于分析监管问题整改的跨层级传导效率。
3. 可与专项问题、审计问题进行对比，形成问题来源结构分析。

### 3.4 审计问题整改

审计问题整改以审计台账和整改信息为核心，更强调审计报告、审计年度、审计部门、整改类型、责任类型等字段。

主要功能点：

| 功能点 | 说明 |
|---|---|
| 台账建立 | 建立审计问题整改台账，支持分发、撤销、导入、导出 |
| 问题下发 | 责任处下发、市财政下发、区财政下发 |
| 整改填报 | 整改信息填报，包含整改措施、整改进展、下一步打算、佐证资料 |
| 汇总上报 | 市财政汇总上报、区县汇总上报 |
| 审核复核 | 责任处审核、评价局复核 |
| 评价局督办 | 对未完成或重点问题进行督办 |

字段特征：

| 类别 | 字段示例 |
|---|---|
| 审计基础信息 | 审计报告名称、审计年度、审计部门、财政区划 |
| 整改属性 | 整改类型、问题金额、责任类型、责任单位、整改状态、整改时限 |
| 责任落实 | 责任厅领导、整改责任处室、整改责任人 |
| 审核销号 | 审计部门是否认可销号、审计部门认可意见、是否销号 |

评价价值：

1. 可评价审计问题整改落实率和审计认可销号情况。
2. 可评价责任处室和责任人承接整改任务的完整性。
3. 可追踪审计问题从下发、填报、汇总、审核到督办的全流程效率。

### 3.5 案例管理

案例管理用于将有代表性的已整改问题沉淀为典型案例或一般案例。

主要功能点：

| 功能点 | 说明 |
|---|---|
| 典型案例申报 | 支持关联已销号问题，填写案例号、案例标题、申报机构、申报人、案例来源、案例领域、案例说明 |
| 案例审核 | 支持通过、退回、填写退回原因 |
| 案例发布 | 将审核通过的案例发布沉淀 |
| 案例明细查询 | 查询案例文号、申报机构、案例领域、案例标题、发布时间、关联问题、材料附件 |

评价价值：

1. 可评价整改成果是否形成可复用经验。
2. 可识别高频问题领域中的典型治理样本。
3. 可作为专项监督评价中的“成果沉淀”和“以案促改”指标来源。

## 5. 专项监督评价体系指标建议

### 5.1 一级指标域建议

建议将问题库指标放入专项监督评价体系的五个一级指标域：

| 一级指标域 | 评价重点 | 建议权重 |
|---|---|---:|
| 问题发现与风险暴露 | 是否发现问题、问题金额和领域集中度 | 20% |
| 责任分解与流程合规 | 是否及时分发、责任是否明确、流程是否规范 | 20% |
| 整改进度与整改成效 | 是否按期整改、金额整改是否到位、是否销号 | 30% |
| 督办问责与闭环质量 | 是否及时督办、是否超期、是否退回、是否问责 | 20% |
| 成果沉淀与治理提升 | 是否形成典型案例、制度完善和经验复用 | 10% |

权重可按专项目标调整。如果专项更强调风险发现，可提高“问题发现与风险暴露”；如果专项进入整改后期，应提高“整改进度与整改成效”和“督办问责与闭环质量”。

### 5.2 指标池

| 指标名称 | 指标口径 | 指标方向 | 适用评价场景 |
|---|---|---|---|
| 问题发现数量 | 某专项、区划、单位、领域下的问题数量 | 中性 | 衡量问题暴露规模，需结合检查覆盖面解读 |
| 问题金额 | 问题类型及金额字段合计 | 负向 | 衡量风险金额规模 |
| 单位覆盖率 | 涉及单位数 / 应覆盖单位数 | 正向 | 衡量专项监督覆盖范围 |
| 问题来源结构 | 自查问题数、复查问题数占比 | 结构型 | 判断问题发现来源 |
| 自查发现率 | 自查发现问题数 / 问题总数 | 正向 | 衡量主动发现问题能力 |
| 复查发现率 | 复查发现问题数 / 问题总数 | 需研判 | 过高可能说明自查不充分 |
| 高风险问题占比 | 高风险问题类型数量或金额 / 问题总量或总金额 | 负向 | 衡量专项风险集中度 |
| 重复问题占比 | 同单位、同领域、同类型重复问题数 / 问题总数 | 负向 | 衡量问题反复发生程度 |
| 清单分发及时率 | 在规定时限内完成分发的问题清单数 / 应分发清单数 | 正向 | 衡量任务传导效率 |
| 责任落实完整率 | 已配置责任单位、责任处室、责任人、责任领导的问题数 / 问题总数 | 正向 | 衡量责任链条完整性 |
| 台账字段完整率 | 必填字段完整的问题数 / 问题总数 | 正向 | 衡量数据质量 |
| 送审及时率 | 在规定时限内送审的问题数 / 应送审问题数 | 正向 | 衡量责任单位响应效率 |
| 审核通过率 | 审核通过数 / 送审数 | 正向 | 衡量整改填报质量 |
| 审核退回率 | 被退回数 / 送审数 | 负向 | 衡量填报质量和审核一次通过情况 |
| 整改完成率 | 已办结或已销号问题数 / 问题总数 | 正向 | 核心整改成效指标 |
| 按期整改率 | 在整改期限内完成的问题数 / 到期应完成问题数 | 正向 | 衡量整改时效 |
| 超期整改率 | 超期未完成或超期完成问题数 / 到期应完成问题数 | 负向 | 衡量整改风险 |
| 已整改金额占比 | 已整改金额 / 问题金额 | 正向 | 衡量资金整改成效 |
| 正在整改金额占比 | 正在整改金额 / 问题金额 | 中性偏负 | 衡量仍在整改的资金规模 |
| 佐证材料上传率 | 已上传佐证材料的问题数 / 需上传佐证的问题数 | 正向 | 衡量整改佐证完整性 |
| 销号率 | 已销号问题数 / 问题总数 | 正向 | 衡量闭环完成情况 |
| 未销号问题占比 | 未销号问题数 / 问题总数 | 负向 | 衡量存量整改压力 |
| 长期未销号率 | 超过阈值仍未销号问题数 / 未销号问题数 | 负向 | 识别历史遗留和难点问题 |
| 督办覆盖率 | 已督办问题数 / 应督办问题数 | 正向 | 衡量监督推动力度 |
| 平均督办轮次 | 督办次数总和 / 已督办问题数 | 需研判 | 轮次高可能说明重视，也可能说明整改难 |
| 督办后完成率 | 督办后完成或销号问题数 / 已督办问题数 | 正向 | 衡量督办有效性 |
| 重新督办率 | 重新督办问题数 / 已督办问题数 | 负向 | 衡量整改反复或督办闭环不足 |
| 问责启动率 | 已记录问责的问题数 / 应问责问题数 | 正向 | 衡量责任追究落实 |
| 问责处理覆盖率 | 有被问责单位或个人的问题数 / 问责问题数 | 正向 | 衡量问责信息完整性 |
| 典型案例转化率 | 关联已销号问题形成案例数 / 已销号问题数 | 正向 | 衡量以案促改成果 |
| 案例审核通过率 | 审核通过案例数 / 申报案例数 | 正向 | 衡量案例质量 |
| 案例发布率 | 已发布案例数 / 审核通过案例数 | 正向 | 衡量成果沉淀落地 |

### 5.3 推荐优先落地指标

如果评价体系第一阶段需要快速落地，建议优先选以下 12 个指标：

| 序号 | 指标名称 | 原因 |
|---:|---|---|
| 1 | 问题发现数量 | 最基础的风险暴露指标 |
| 2 | 问题金额 | 衡量问题严重程度 |
| 3 | 自查发现率 | 体现主动监督能力 |
| 4 | 责任落实完整率 | 体现问题是否有人负责 |
| 5 | 送审及时率 | 体现责任单位响应效率 |
| 6 | 审核退回率 | 体现整改填报质量 |
| 7 | 整改完成率 | 核心成效指标 |
| 8 | 按期整改率 | 核心时效指标 |
| 9 | 超期整改率 | 核心风险预警指标 |
| 10 | 已整改金额占比 | 衡量资金整改成效 |
| 11 | 佐证材料上传率 | 衡量整改证据链完整性 |
| 12 | 销号率 | 衡量闭环完成程度 |

第二阶段可增加督办、问责、案例转化类指标，使评价从“是否整改完成”扩展到“整改是否真实有效、是否形成长效治理”。

## 6. 专项监督评分模型建议

可采用 100 分制，将问题库指标转化为专项监督评价得分：

| 评分模块 | 分值 | 推荐指标 |
|---|---:|---|
| 问题发现与覆盖 | 20 | 单位覆盖率、自查发现率、问题来源结构、问题类型覆盖 |
| 风险严重程度 | 15 | 问题金额、高风险问题占比、重复问题占比 |
| 整改落实 | 30 | 整改完成率、已整改金额占比、按期整改率、销号率 |
| 过程质量 | 20 | 责任落实完整率、送审及时率、审核退回率、佐证材料上传率 |
| 督办与治理提升 | 15 | 督办后完成率、超期整改率、问责启动率、典型案例转化率 |

评分处理建议：

1. 对“问题金额、超期整改率、审核退回率、重复问题占比”等负向指标采用反向计分。
2. 对“问题发现数量”不建议直接正向或负向计分，应结合覆盖率、问题金额、复查发现率进行解释。
3. 对“平均督办轮次”不建议单独打分，应与督办后完成率、超期整改率一起判断。
4. 对“自查发现率”建议设合理区间，过低说明主动发现不足，过高但整改差也不能高分。

## 7. 可用于看板分析的主题

如果后续要做专项监督评价看板，可以围绕以下主题组织：

| 看板主题 | 核心问题 |
|---|---|
| 问题总览 | 哪些专项、区划、单位问题最多，金额最大 |
| 风险领域分析 | 哪些问题类型高发，哪些领域金额集中 |
| 整改进度分析 | 哪些问题已整改、正在整改、未开始、超期 |
| 责任落实分析 | 哪些责任单位、处室、责任人任务多、退回多、超期多 |
| 督办成效分析 | 督办后是否推动完成，哪些问题反复督办仍未销号 |
| 佐证材料分析 | 哪些问题缺少佐证，哪些附件不完整 |
| 销号质量分析 | 销号率、审计认可销号、审核通过、退回原因 |
| 案例沉淀分析 | 哪些已销号问题形成典型案例，案例领域分布如何 |

## 8. 数据口径风险与后续确认点

本次分析基于菜单、路由、页面字段和接口命名，能够判断业务功能和可用指标方向，但尚未读取数据库表结构和真实业务数据样本。落地评价体系前建议确认：

1. 问题总数的统计粒度：按问题记录、按责任单位拆分记录，还是按问题清单明细。
2. 问题金额口径：合计金额是否等于各问题类型金额之和，是否存在万元、元等单位差异。
3. 销号口径：已办结、已审核、已销号、审计部门认可销号之间的最终关系。
4. 超期口径：按整改截止时间、预计完成时间，还是按督办轮次期限判断。
5. 督办对象：哪些状态或期限的问题应纳入“应督办”分母。
6. 问责口径：是否只有严重问题需要问责，问责信息是否必填。
7. 典型案例口径：是否必须关联已销号问题，典型案例和一般案例是否分别计分。
8. 角色权限差异：评价局、市财政、区县、省直、牵头处、责任处看到的数据范围可能不同，评价口径应统一到后台统计权限。

## 9. 最终建议

专项监督评价体系应把问题库定位为“问题整改闭环数据源”，而不是简单的问题列表。第一阶段建议围绕“发现、分发、整改、审核、销号”建立基础评价指标；第二阶段再叠加“督办、问责、案例、制度完善”等治理提升指标。

最值得优先纳入评价体系的指标是：

1. 整改完成率
2. 按期整改率
3. 超期整改率
4. 已整改金额占比
5. 销号率
6. 责任落实完整率
7. 审核退回率
8. 佐证材料上传率
9. 自查发现率
10. 典型案例转化率

这些指标既能反映问题风险，也能反映整改推进、责任落实和治理成效，适合作为专项监督评价体系的基础指标池。
